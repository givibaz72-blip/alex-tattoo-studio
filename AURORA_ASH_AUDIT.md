# Aurora & Ash — UX/UI аудит и план улучшений

_Дата: 10 мая 2026. Стек: Next.js 16, React 19, Payload CMS 3, Tailwind v4, Framer Motion 12._

## Сводка
Проект имеет сильный визуальный язык (золото на угольном фоне, Playfair Display, полноэкранное меню). Технический стек современный. Основные потери — в связности конверсионных путей (меню «Inquiry» уводит к футеру, а не на форму), в нескольких битых ссылках, в избыточной зависимости от hover-only состояний и в эстетических, но недосчитанных деталях формы заявки. Ниже — поименный план по каждой области.

---

## 1. Визуальный дизайн

### 1.1 Иерархия золотого через альфа-каналы ломает контраст
- **Текущее:** `#D4AF37` на `#121212` → 7.5:1, но `text-[#D4AF37]/40..60` опускает small-text до 3.0–4.2:1.
- **Проблема:** `.eyebrow` (11 px) и `.label-line` (13 px) не проходят WCAG AA.
- **Решение:** В `globals.css` добавить `--text-primary` (D4AF37 / 1.0), `--text-secondary` (B89530), `--text-muted` (8A7B45). Заменить все `/40..60` на эти три токена.
- **Приоритет: высокий.**

### 1.2 Footer ломает визуальный язык сайта
- **Текущее:** `Footer.tsx` использует `#fff`, `--neutral-300`. Остальной сайт — золотая палитра.
- **Решение:** Заголовки footer'а — `--color-gold`, тело — off-white `#E8DCB6`. Убрать `text-neutral-300` из `SocialLinks` (footer-вариант).
- **Приоритет: высокий.**

### 1.3 Нет единой типографической шкалы
- **Текущее:** Размеры разбросаны как Tailwind-классы по компонентам.
- **Решение:** Завести `--text-display`, `--text-h1`, `--text-h2`, `--text-lead` в `@theme inline`, плюс утилиты `.t-display`, `.t-h1` и т.д. Перевести Hero/About/Team/FeaturedWorks на них.
- **Приоритет: средний.**

### 1.4 Hero `&` выровнен вручную через `align-middle`
- **Текущее:** `<span className="... align-middle">&</span>`.
- **Решение:** `inline-block; transform: translateY(-0.08em); vertical-align: baseline`. Размер 0.55em от h1.
- **Приоритет: низкий.**

## 2. Навигация и информационная архитектура

### 2.1 «Inquiry» из меню уходит к футеру, а не на форму
- **Текущее:** `NavMenu` Inquiry → `#contact` (footer). Hero CTA → `/inquiry`.
- **Решение:** В `NavMenu` поменять Inquiry-CTA на `href="/inquiry"`. ID футера переименовать с `contact` на `footer`.
- **Приоритет: высокий — прямое влияние на конверсию.**

### 2.2 Битые ссылки в footer и menu
- **Текущее:** `/contact`, `/accessibility`, `/terms`, `/aftercare`, `/faq` — маршруты отсутствуют (есть только `inquiry`, `portfolio`, `privacy`, `[slug]`).
- **Решение:** Создать в Pages-коллекции CMS соответствующие документы либо удалить ссылки. Добавить build-time-проверку всех Link-href.
- **Приоритет: высокий.**

### 2.3 Полноэкранное меню скрывает всю навигацию
- **Текущее:** Только «MENU» в шапке.
- **Решение:** Слева — логотип; справа — постоянный «Inquiry» + «Menu». На скролле Inquiry приглушается до иконки.
- **Приоритет: высокий.**

### 2.4 Нет хлебных крошек на странице артиста
- **Решение:** `Home / Artists / {Имя}` под header'ом, с JSON-LD `BreadcrumbList`. «Back» оставить отдельной кнопкой.
- **Приоритет: средний.**

### 2.5 Логотип нечитаем на светлых hero-кадрах
- **Решение:** Добавить градиент `from-[#0a0a0a]/40 to-transparent` под header'ом сразу при `atTop`.
- **Приоритет: средний.**

## 3. Интерактивные элементы и анимация

### 3.1 Hero-параллакс на `position: fixed` ломается на iOS
- **Решение:** Заменить `fixed` на `absolute inset-0` внутри hero-секции. Параллакс — через `useScroll({ target: heroRef, offset: ['start start', 'end start'] })`.
- **Приоритет: средний.**

### 3.2 Hover-only подписи в `WorksGallery`
- **Текущее:** `figcaption opacity-0 group-hover:opacity-100`. На тач-устройствах не видно.
- **Решение:** На `@media (hover: none)` показывать caption всегда (opacity 0.7).
- **Приоритет: высокий.**

### 3.3 Lightbox без свайпа и без preload
- **Решение:** Touch-handler / Framer Motion drag со порогом 50 px. Preload следующего изображения через `new Image().src = nextUrl`. На ≤ 480 px вынести Close в центр сверху, стрелки — вниз.
- **Приоритет: средний.**

### 3.4 `whileInView` без проверки `prefers-reduced-motion`
- **Решение:** Добавить `useReducedMotion()` в `TeamGrid`, `AccordionBlock`, `ParallaxSection` и обнулять transition.
- **Приоритет: средний.**

### 3.5 Тач-цели меньше 44×44
- **Решение:** Всем кнопкам/ссылкам — `min-h-11 min-w-11` (Tailwind).
- **Приоритет: средний.**

### 3.6 Layout-shift при выборе артиста
- **Решение:** Заменить `scale-105` на rim-light: `border-[#D4AF37] + outline outline-2 outline-offset-4 outline-[#D4AF37] + shadow-[0_0_30px_rgba(212,175,55,0.25)]`.
- **Приоритет: низкий.**

## 4. Функциональность бронирования (`/inquiry`)

### 4.1 «Telegram or phone» — два смысла в одном инпуте
- **Решение:** Радио `Telegram | Phone | Email only`, динамический input с правильным `type/inputmode/autoComplete` и валидацией. В payload — два отдельных поля.
- **Приоритет: высокий.**

### 4.2 Email — единственный обязательный канал
- **Решение:** «Хотя бы один из» email/telegram/phone. Серверная проверка повторяет клиентскую.
- **Приоритет: высокий.**

### 4.3 Нет шага подтверждения/превью
- **Решение:** Добавить шаг «Review» — компактная сводка с inline-edit. Прогресс-бар на 5 делений.
- **Приоритет: высокий.**

### 4.4 Нет защиты от спама
- **Решение:** Honeypot-поле + Cloudflare Turnstile + rate-limit per IP (Upstash/Vercel KV).
- **Приоритет: высокий.**

### 4.5 Несогласованные email-адреса
- **Текущее:** `hello@auroraash.com` (footer) vs `studio@aurora-ash.com` (inquiry error).
- **Решение:** Один источник правды — `siteSettings.email` в CMS. Удалить хардкод.
- **Приоритет: высокий.**

### 4.6 «Спасибо» без следующего шага
- **Решение:** На success-экране — vCard, IG-deep-link, и реальное обещание ответа в час через Telegram-бот студии (`afterChange` hook в Payload).
- **Приоритет: средний (высокий ROI).**

### 4.7 Reference links без валидации, без загрузки изображений
- **Решение:** Парсить ввод как URL-список; добавить drag-and-drop загрузки 3–6 файлов ≤ 5 МБ.
- **Приоритет: средний.**

### 4.8 Прогресс-бар без меток
- **Решение:** Лейблы `Artist › Vision › Details › Contact`. На мобильном — `Step 2 of 4 · Vision`.
- **Приоритет: средний.**

### 4.9 Inline-валидация только при попытке шага
- **Решение:** `onBlur` per field; кнопка Next динамически `disabled` или показывает все ошибки разом (текущее поведение оставить как fallback).
- **Приоритет: средний.**

### 4.10 Бюджет на одном экране с placement+size
- **Решение:** Вынести в отдельный шаг или в аккордеон с pre-selected «Open to discuss».
- **Приоритет: низкий.**

## 5. Производительность и доступность

### 5.1 Hero LCP не оптимизирован
- **Решение:** В `siteSettings.heroImage` — три size'а (768/1280/1920); правильный `sizes`-атрибут; `images.formats: ['image/avif', 'image/webp']` в `next.config`.
- **Приоритет: средний.**

### 5.2 Скролл-локи не компенсируют скролл-бар
- **Решение:** Перед локом сохранить `paddingRight = innerWidth - clientWidth`, снять при разлоке.
- **Приоритет: средний.**

### 5.3 Skip link может оказаться за header'ом
- **Решение:** `z-[200]`, `top: 1rem`, добавить `box-shadow` для контраста.
- **Приоритет: низкий.**

### 5.4 Suspense без fallback
- **Решение:** Mini-skeleton для NavBar (72 px + логотип); spinner с прогресс-баром для InquiryForm.
- **Приоритет: низкий.**

### 5.5 Framer Motion загружается на каждый клиентский компонент
- **Решение:** Lightbox в `WorksGallery` — через `next/dynamic({ ssr: false })`. Hero-появления — на чистом CSS keyframes.
- **Приоритет: низкий.**

### 5.6 SEO/Open Graph почти отсутствуют
- **Решение:** В `app/(frontend)/layout.tsx` добавить `metadata.openGraph`, `metadata.twitter`, `metadataBase`, `alternates.languages`. Для артистов — динамический OG через `app/opengraph-image.tsx` (Next `ImageResponse`).
- **Приоритет: высокий.**
